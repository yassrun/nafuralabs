8. Contrats employés + habilitations (CACES, SST, BE/BR, …).

### B-RH-02 — Pointage chantier (batch + multi-pointage équipe)

**Goal :** entités `Pointage` + `PointageBatch` permettant la saisie batch (1 chef pointe 10 ouvriers en 1 minute).

**Endpoints :**
```
GET    /api/v1/rh/pointages?chantierId=...&date=2026-05-13
GET    /api/v1/rh/pointages/by-employe?employeId=...&from=...&to=...
POST   /api/v1/rh/pointage-batches                      ← création batch (1 chef + N employés + photo + signature)
POST   /api/v1/rh/pointage-batches/{id}/valider         ← validation par hiérarchie
PUT    /api/v1/rh/pointages/{id}
GET    /api/v1/chantiers/{id}/pointages/synthese?from=...&to=... ← agrégat budget chantier
```

**Champs `PointageBatch` :**
```
id, tenant_id, chef_user_id, chantier_id, date_pointage,
photo_url (optionnel), signature_url (chef), gps_lat, gps_lng,
status (BROUILLON/SOUMIS/VALIDE/REJETE),
created_at
```

**Champs `Pointage` :**
```
id, tenant_id, batch_id, employe_id, date,
heures_normales, heures_sup_25, heures_sup_50, heures_sup_100,
chantier_id, poste_budgetaire_id,
created_at, updated_at
```

**Logique offline-sync :** le frontend (cf. roadmap Round 2 Task 15 M-MOB-02) garde les batches en IndexedDB pendant 7 jours et les synchronise via `POST /pointage-batches` quand connectivité revient. Le backend doit :

- accepter un `clientId` (UUID généré côté mobile) pour idempotence.
- répondre `409 Conflict` avec body si `clientId` déjà existant.

**Désinjection :**
- `pages/rh/pointage/services/pointage-mock.service.ts` → renommer `pointage-api.service.ts` (pure HTTP)
- `pages/rh/pointage/pointage-listing/`, `pointage-saisie/`, `pointage-validation/`

**Effort :** 3-4 j.h

### B-RH-05 — Fiches de paie (génération + storage)

**Goal :** entités `FichePaie` + `LignePaie` avec calcul MA descendu côté backend.

**Logique calcul (à porter depuis `web/app/applications/erp/rh/services/paie-engine.service.ts`) :**

```
salaire_brut = salaire_base + heures_sup + primes
cnss_salarial = MIN(salaire_brut, 6000) * 4.48%
amo_salarial = salaire_brut * 2.26%
cimr_salarial = salaire_brut * cimr_taux  ← si cadre
ipe_salarial = salaire_brut * 0.19%
revenu_imposable = salaire_brut - cnss - amo - cimr - ipe - frais_pro_20pct
igr = bareme_progressif(revenu_imposable)
net_a_payer = salaire_brut - cnss - amo - cimr - ipe - igr
```

**Endpoints :**
```
POST   /api/v1/rh/fiches-paie/generate?mois=2026-05&societeId=...   ← génération batch
GET    /api/v1/rh/fiches-paie?employeId=...&mois=...
GET    /api/v1/rh/fiches-paie/{id}/pdf
POST   /api/v1/rh/fiches-paie/{id}/valider
POST   /api/v1/rh/fiches-paie/{id}/comptabiliser
```

**Déclarations :**
```
GET /api/v1/rh/declarations/damancom-xml?mois=2026-05      ← XML BAP
GET /api/v1/rh/declarations/igr-etat-9421?annee=2026       ← état IGR
GET /api/v1/rh/declarations/etat-1208?annee=2026           ← état 1208
GET /api/v1/rh/declarations/retenue-source?trimestre=Q1
```

**Tests unitaires obligatoires :** `PaieEngineServiceTest` (10+ cas : CDI cadre, CDI non-cadre, HS, congés non pris, …).

**Désinjection :** `pages/rh/paie/**` (services + 4 pages déclarations).

**Effort :** 2-3 j.h

**Règles :**


**Source :** détection automatique depuis `Pointage` (cf. B-RH-02) à la validation du batch.

**Effort :** 1-2 j.h
